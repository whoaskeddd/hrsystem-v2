# HR Platform

## Local Run Without Docker

### 1. Backend

```powershell
cd C:\develop\hrsystem-v2\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Create `backend\.env` with this minimal config:

```env
APP_NAME=HR Platform API
DEBUG=true
SECRET_KEY=dev-secret-key
DATABASE_URL=sqlite:///./hrplatform.db
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173,http://0.0.0.0:5173
CORS_ORIGIN_REGEX=^https?://(localhost|127\.0\.0\.1|0\.0\.0\.0|192\.168\.\d{1,3}\.\d{1,3}|10\.\d{1,3}\.\d{1,3}\.\d{1,3}|172\.(1[6-9]|2\d|3[0-1])\.\d{1,3}\.\d{1,3})(:\d+)?$
```

Start backend:

```powershell
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2. Frontend

```powershell
cd C:\develop\hrsystem-v2\frontend
npm install
npm run dev
```

`npm run dev` already starts Vite on `0.0.0.0:5173`, so no extra `--host` flag is needed.

If you open it from another device, set backend URL before start:

```powershell
$env:VITE_API_BASE_URL="http://YOUR_PC_IP:8000/api/v1"
npm run dev
```

If `VITE_API_BASE_URL` is not set, frontend automatically uses:

```text
http://<current-browser-host>:8000/api/v1
```

Special case: if browser is opened on a service address like `198.18.x.x` or `198.19.x.x`, frontend falls back to `http://localhost:8000/api/v1`.


Платформа для поиска работы и подбора персонала в стиле hh.ru, с backend на FastAPI и frontend на React + Tailwind CSS.

## Общие сведения

Проект представляет собой web-платформу для поиска работы и подбора персонала, где есть две основные стороны: соискатель и работодатель. Платформа поддерживает публикацию вакансий, создание резюме, отклики, поиск, фильтрацию, чат, уведомления и базовые HR-инструменты, аналогичные core-сценариям hh.ru.

Цель проекта — дать пользователю удобный и быстрый сервис для поиска работы и найма сотрудников, с современным интерфейсом, адаптивной версткой, безопасной авторизацией и масштабируемой серверной архитектурой на FastAPI.

## Роли пользователей

Система поддерживает следующие роли:

- Гость.
- Соискатель.
- Работодатель.
- HR-менеджер.
- Администратор.

Гость может просматривать часть вакансий и общую информацию, но не может откликаться и создавать контент. Соискатель создает резюме, ищет вакансии, откликается, сохраняет вакансии и подписки. Работодатель публикует вакансии, просматривает отклики, общается с кандидатами и управляет подбором. Администратор управляет пользователями, модерацией, справочниками и настройками платформы.

## Основной функционал

### Для соискателя

- Регистрация и вход.
- Создание и редактирование резюме.
- Публикация нескольких резюме.
- Выбор видимости резюме: открытое, скрытое, по ссылке, только для работодателей.
- Поиск вакансий по ключевым словам, зарплате, опыту, занятости, формату работы, локации.
- Сохранение вакансий в избранное.
- Отклик на вакансию.
- Сопроводительное письмо.
- Подписка на автопоиск и уведомления о новых вакансиях.
- История откликов и статусы по ним.
- Чат с работодателем.
- Просмотр рекомендаций вакансий.
- Настройка профиля и приватности.

### Для работодателя

- Регистрация компании.
- Верификация компании.
- Создание и редактирование вакансий.
- Черновик, публикация, архивирование вакансий.
- Поиск кандидатов по базе резюме.
- Фильтры по опыту, навыкам, зарплате, образованию, локации.
- Просмотр откликов на вакансии.
- Приглашение кандидата.
- Отклонение кандидата.
- Чат и обмен сообщениями.
- Ведение воронки подбора по стадиям.
- Базовая аналитика: просмотры вакансии, отклики, конверсия, время закрытия вакансий.

### Для администратора

- Управление пользователями и компаниями.
- Модерация вакансий и резюме.
- Блокировка и разблокировка аккаунтов.
- Управление справочниками.
- Просмотр логов и событий.
- Управление жалобами.
- Настройка категорий, регионов, типов занятости, навыков и ролей.

## Функции поиска

Поиск является одним из ключевых модулей. Для вакансий и резюме доступны:

- Поиск по тексту.
- Фильтрация по региону и городу.
- Фильтрация по зарплате.
- Фильтрация по опыту.
- Фильтрация по графику работы.
- Фильтрация по типу занятости.
- Фильтрация по уровню образования.
- Фильтрация по удаленной работе.
- Сортировка по релевантности, дате публикации, зарплате и популярности.

Система поддерживает сохраненные поиски и подписки на новые результаты, как в мобильной и веб-логике hh.ru.

## Чат и уведомления

В платформе есть встроенный чат между соискателем и работодателем. Чат поддерживает:

- Текстовые сообщения.
- Системные сообщения о смене статуса отклика.
- Уведомления о новых сообщениях.
- Уведомления о приглашениях, откликах и изменении статуса заявки.

Также предусмотрены:

- Push-уведомления.
- Email-уведомления.
- In-app уведомления.

Пользователь управляет подписками и типами уведомлений из личного кабинета.

## Мессенджер и звонки

Платформа включает встроенный мессенджер и голосовые звонки внутри системы. Чат должен быть связан с вакансией, откликом, приглашением или профилем компании. Для real-time обмена сообщениями подходит WebSocket, который поддерживается FastAPI.

### Чат должен поддерживать

- Личные диалоги.
- Диалоги по конкретной вакансии.
- Системные сообщения.
- Индикатор прочтения.
- Статусы доставки.
- Вложения: файлы, изображения, резюме.
- Закреплённые сообщения.
- Поиск по истории переписки.
- Блокировку и жалобу на собеседника.

### Звонки должны поддерживать

- Входящие и исходящие голосовые звонки.
- Звонки из чата.
- Звонки из карточки кандидата.
- Звонки из карточки вакансии.
- Уведомление о входящем звонке.
- Маркировку звонка как HR-звонка.
- Историю звонков.
- Запись звонка, если это разрешено.
- Текстовую расшифровку звонка.
- Краткий итог разговора.
- Статусы: не дозвонились, разговор состоялся, кандидат перезвонит, отказ.

Для звонков технически целесообразно использовать WebRTC, а сигнализацию — через FastAPI WebSocket или отдельный signaling layer.

## Интерфейс и дизайн

Дизайн должен быть минималистичным, чистым, легким, с ощущением премиальности и спокойствия. Основная визуальная идея — темная тема с золотыми акцентами, без перегрузки декоративными элементами. OpenAI-стилистика должна выражаться в лаконичности, большом количестве воздуха, мягких тенях, нейтральной типографике и аккуратных контрастах.

### Рекомендуемая палитра

- Фон: почти черный, графитовый или глубокий угольно-серый.
- Поверхности: темно-серые панели.
- Акценты: мягкое золото, шампань, приглушенный янтарный.
- Текст: светло-серый и белый для основного контента.

### UI-принципы

- Много свободного пространства.
- Карточки с мягкими скруглениями.
- Один главный акцентный цвет.
- Минимум тяжелых градиентов.
- Плавные hover-эффекты.
- Легкие тени.
- Четкая визуальная иерархия.

## Анимации

Анимации должны быть аккуратными, ненавязчивыми и ускорять восприятие интерфейса. Во React лучше использовать CSS-анимации и библиотеки для плавных transitions, чтобы обеспечить мягкие появления модалок, dropdown-меню, карточек и сайдбаров.

### Рекомендуемые анимации

- Появление карточек вакансий и резюме.
- Плавное раскрытие фильтров.
- Анимация модальных окон.
- Hover-подсветка карточек.
- Плавная загрузка skeleton-элементов.
- Переключение вкладок с fade/slide.
- Анимация отправки сообщения в чате.
- Мягкий hover на кнопках и иконках.
- Плавный drawer/sidebar на мобильных устройствах.

### Ограничения

- Без резких масштабирований.
- Без чрезмерных bounce-эффектов.
- Длительность большинства анимаций: 150–250 мс.
- Использовать easing, близкий к standard UI motion.
