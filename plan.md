# plan.md

## Project
HR Platform  
Stack: **FastAPI + React + Tailwind CSS**  
Roles: **Backend Developer**, **Frontend Developer**

---

## 1. Goal

Сделать HR-платформу для поиска работы и подбора персонала в стиле hh.ru, с:
- вакансиями,
- резюме,
- откликами,
- поиском и фильтрами,
- чатами,
- уведомлениями,
- мессенджером,
- голосовыми звонками,
- админ-панелью,
- минималистичным dark UI в тёмно-золотой палитре.

---

## 2. Team Responsibilities

### Backend Developer
Отвечает за:
- архитектуру FastAPI,
- БД и модели,
- авторизацию и роли,
- API для всех сущностей,
- WebSocket для чата,
- signaling для звонков,
- уведомления,
- админские эндпоинты,
- тесты backend-логики,
- документацию API.

### Frontend Developer
Отвечает за:
- React приложение,
- роутинг и state management,
- UI-компоненты,
- адаптивную верстку,
- страницы пользователей и работодателей,
- чат-интерфейс,
- UI звонков,
- анимации,
- dark theme,
- интеграцию с API,
- frontend тесты.

## 3. Milestone Plan

### Milestone 1 — Foundation
Цель: поднять каркас проекта и базовую архитектуру.

#### Backend
- Создать структуру проекта FastAPI.
- Настроить окружение, конфиги, логирование.
- Подключить БД.
- Настроить ORM и миграции.
- Подготовить базовые схемы пользователей и ролей.
- Настроить JWT auth.

#### Frontend
- Создать React-проект.
- Подключить router, store, HTTP-клиент.
- Настроить структуру папок.
- Сверстать layout приложения.
- Подготовить тему и дизайн-систему.
- Сделать базовые компоненты: button, input, modal, card.

#### Результат
- Проект запускается локально.
- Есть единая структура.
- Есть входная страница и базовый дизайн.

## 5. Parallel Work Rules

- Backend и frontend начинают с общей модели данных.
- Все ключевые сущности должны иметь согласованные DTO/типы.
- Контракт API фиксируется до начала интеграции.
- Для каждой новой фичи сначала согласуется response/request schema.
- Реализацию чата и звонков лучше делать отдельными спринтами.
- Каждый интерфейс должен иметь mock-data до готовности API.

## 7. Definition of Done

Фича считается готовой, если:
- backend API реализован,
- frontend экран сделан,
- данные корректно отображаются,
- есть обработка ошибок,
- есть loading states,
- есть базовые тесты,
- компонент адаптивен,
- код проходит линтер,
- документация обновлена.
