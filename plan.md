# plan.md

## Project
HR Platform  
Stack: **FastAPI + React + Tailwind CSS**  
Roles: **Backend Developer**, **Frontend Developer**

---

## 1. Goal

Сделать HR-платформу для поиска работы и подбора персонала в стиле hh.ru, с:
- вакансиями,
- резюме,
- откликами,
- поиском и фильтрами,
- чатами,
- уведомлениями,
- мессенджером,
- голосовыми звонками,
- админ-панелью,
- минималистичным dark UI в тёмно-золотой палитре.

---

## 2. Team Responsibilities

### Backend Developer
Отвечает за:
- архитектуру FastAPI,
- БД и модели,
- авторизацию и роли,
- API для всех сущностей,
- WebSocket для чата,
- signaling для звонков,
- уведомления,
- админские эндпоинты,
- тесты backend-логики,
- документацию API.

### Frontend Developer
Отвечает за:
- React приложение,
- роутинг и state management,
- UI-компоненты,
- адаптивную верстку,
- страницы пользователей и работодателей,
- чат-интерфейс,
- UI звонков,
- анимации,
- dark theme,
- интеграцию с API,
- frontend тесты.

---

## 3. Milestone Plan

### Milestone 1 — Foundation
Цель: поднять каркас проекта и базовую архитектуру.

#### Backend
- Создать структуру проекта FastAPI.
- Настроить окружение, конфиги, логирование.
- Подключить БД.
- Настроить ORM и миграции.
- Подготовить базовые схемы пользователей и ролей.
- Настроить JWT auth.

#### Frontend
- Создать React-проект.
- Подключить router, store, HTTP-клиент.
- Настроить структуру папок.
- Сверстать layout приложения.
- Подготовить тему и дизайн-систему.
- Сделать базовые компоненты: button, input, modal, card.

#### Результат
- Проект запускается локально.
- Есть единая структура.
- Есть входная страница и базовый дизайн.

---

### Milestone 2 — Authentication and Profiles
Цель: реализовать регистрацию, вход и личные кабинеты.

#### Backend
- Регистрация пользователя.
- Логин/логаут.
- Обновление токена.
- Восстановление пароля.
- Профиль соискателя.
- Профиль работодателя.
- Ролевой доступ.
- Механизм верификации компании.

#### Frontend
- Страницы регистрации и входа.
- Страница восстановления пароля.
- Личный кабинет соискателя.
- Личный кабинет работодателя.
- Форма редактирования профиля.
- Защищённые маршруты.
- UI состояния: loading, error, success.

#### Результат
- Пользователь может зарегистрироваться, войти и попасть в свой кабинет.
- Роли разделены.

---

### Milestone 3 — Vacancies and Resumes
Цель: ядро платформы.

#### Backend
- CRUD вакансий.
- CRUD резюме.
- Связь vacancy ↔ employer.
- Связь resume ↔ candidate.
- Публикация, черновик, архив.
- Видимость резюме.
- Валидация данных.

#### Frontend
- Страница списка вакансий.
- Страница вакансии.
- Форма создания и редактирования вакансии.
- Страница списка резюме.
- Форма создания и редактирования резюме.
- Карточки вакансий и резюме.
- Фильтр видимости резюме.

#### Результат
- Работодатель может публиковать вакансии.
- Соискатель может создавать резюме.
- Данные отображаются в UI.

---

### Milestone 4 — Search and Filters
Цель: поиск по платформе.

#### Backend
- Поиск вакансий.
- Поиск резюме.
- Фильтры по региону, зарплате, опыту, занятости, образованию, удалёнке.
- Сортировка.
- Сохранённые поиски.
- Подписки на автопоиск.

#### Frontend
- Поисковая строка.
- Панель фильтров.
- Сортировка.
- Сохранённые фильтры.
- UI подписки на поиск.
- Пагинация или infinite scroll.

#### Результат
- Пользователь быстро находит вакансии и кандидатов по нужным параметрам.

---

### Milestone 5 — Applications and Favorites
Цель: отклики и избранное.

#### Backend
- Отклик на вакансию.
- Отмена отклика.
- Статусы отклика.
- Избранные вакансии.
- Избранные кандидаты.
- История действий.
- Сопроводительное письмо.

#### Frontend
- Кнопка отклика.
- Форма сопроводительного письма.
